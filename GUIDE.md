# מדריך שימוש — CCSO v3.0
## Claude Code Smart Optimizer

---

## מה זה CCSO?

CCSO הוא שכבת חכמה שיושבת בין אתה לבין Claude Code.
במקום להקליד `claude`, אתה מקליד `ccso` — והכל קורה אוטומטית ברקע.

---

## הפעלה

```bash
ccso          # פתיחת Smart REPL
```

ברגע שנפתח, תראה:
```
╔══════════════════════════════════════╗
║   Claude Code Smart Optimizer (CCSO) ║
║   v3.0.0  —  חוסך טוקנים בשקט       ║
╚══════════════════════════════════════╝

cc>
```

בשורת `cc>` כותבים את הפקודות לקלוד — בדיוק כמו שהיית כותב לקלוד רגיל.

## מה קורה אוטומטית (ללא פעולה מצדך)

### 1. תרגום עברית → אנגלית
כתוב בעברית, קלוד מקבל אנגלית.
```
cc> תבדוק למה כפתור ההתחברות לא עובד
  [CCSO] 🔄 תורגם עברית → אנגלית
[Claude] Found the issue: the onClick handler is missing...
```
**חיסכון: עד 40% בטוקנים**

### 2. ניתוב מודל חכם
CCSO בוחר אוטומטית את המודל הנכון:
- **⚡ Haiku** — שאלות פשוטות (זול פי 10)
- **🎯 Sonnet** — משימות קוד רגילות (ברירת מחדל)
- **🧠 Opus** — ריפקטור מלא, ארכיטקטורה מורכבת
```
cc> מה ההבדל בין map ל-forEach?
  [CCSO] שאלה פשוטה → Haiku (זול יותר)
```

### 3. הסרת נימוסים
"בבקשה", "תודה", "האם תוכל" — מוסרים אוטומטית. פחות טוקנים.

### 4. כיווץ קוד
כשמדביקים קוד בפרומפט — הערות ורווחים מיותרים מוסרים.
**חיסכון: 20–30% על פרומפטים עם קוד**

### 5. הקשר Git אוטומטי
כשכותבים "תקן", "שגיאה", "לא עובד" — CCSO מוסיף אוטומטית:
- `git status` — מה השתנה
- `git diff` — הקוד שהשתנה
- 3 ה-commits האחרונים
```
cc> תקן את הבאג הזה
  [CCSO] 🔀 הקשר Git נוסף אוטומטית
```

### 6. סורק מפתחות API
אם בטעות הדבקת API key בפרומפט — CCSO מזהיר אותך לפני השליחה:
```
  ⚠️  זוהה Anthropic API Key בפרומפט — בדוק שלא חשפת מידע רגיש!
```

### 7. Handoff אוטומטי
כשהסשן מגיע ל-25 פקודות או $0.80 — CCSO:
1. מבקש מקלוד לכתוב `HANDOFF.md` עם סיכום
2. שומר לזיכרון
3. מאפס את הסשן

## פקודות בתוך ה-REPL

| פקודה | מה עושה |
|---|---|
| `/status` | הצגת עלות, פקודות, זמן |
| `/handoff` | שמירת סיכום וניקוי ידני |
| `/history` | הצגת 25 הפרומפטים האחרונים |
| `/history search <מילה>` | חיפוש בהיסטוריה |
| `/memory list` | הצגת זיכרון שמור |
| `/memory add "עובדה"` | שמירת עובדה על הפרויקט |
| `/memory clear` | מחיקת זיכרון הפרויקט |
| `/memory global "עובדה"` | שמירת עובדה לכל הפרויקטים |
| `/template` | הצגת כל התבניות |
| `/template bug` | תבנית לדיווח באג |
| `/template review` | תבנית ל-Code Review |
| `/template refactor` | תבנית ל-Refactor |
| `/template tests` | תבנית לכתיבת טסטים |
| `/template explain` | תבנית להסבר קוד |
| `/template docs` | תבנית לתיעוד |
| `/template optimize` | תבנית לאופטימיזציה |
| `/template security` | תבנית לבדיקת אבטחה |
| `/dashboard` | פתיחת דשבורד בדפדפן |
| `/exit` | יציאה |

## פקודות מחוץ ל-REPL (מהטרמינל הרגיל)

```bash
ccso                         # פתיחת REPL
ccso --init                  # אשף הגדרת פרויקט (CLAUDE.md)
ccso --config                # שינוי הגדרות
ccso --status                # סטטוס מהיר
ccso --dashboard             # פתיחת דשבורד
ccso --uninstall             # הסרת CCSO

ccso template                # רשימת תבניות
ccso template bug            # הצגת תבנית באג
ccso history                 # היסטוריית פרומפטים
ccso history <מילה>          # חיפוש בהיסטוריה

ccso memory list             # הצגת זיכרון
ccso memory add "עובדה"      # הוספה לזיכרון
ccso memory clear            # מחיקת זיכרון הפרויקט
ccso memory global "עובדה"   # זיכרון גלובלי

ccso mcp list                # רשימת MCPs
ccso mcp add github          # הוספת GitHub
ccso mcp add notion          # הוספת Notion
ccso mcp add brave_search    # הוספת חיפוש אינטרנט
```

## זרימת עבודה מומלצת

### תחילת פרויקט חדש
```bash
cd /path/to/my-project
ccso --init        # יוצר CLAUDE.md + .claudeignore
ccso               # מתחיל לעבוד
```

### עבודה יומית
```bash
ccso               # פתיחת REPL
cc> תבדוק את הבאג ב-login.tsx
cc> תוסיף validation לטופס
cc> /status        # לבדוק עלות
cc> /handoff       # בסוף היום — שמירת סיכום
```

### שמירת החלטות חשובות
```
cc> /memory add "החלטנו להשתמש ב-Zustand ולא Redux"
cc> /memory add "ה-API endpoint הוא /api/v2/users"
```
בסשן הבא קלוד יקבל את ההקשר הזה אוטומטית.

### שימוש בתבניות
```
cc> /template bug
[CCSO מציג את תבנית הבאג]
[מעתיקים, ממלאים את הפרטים, ושולחים]
```

- טוקנים שנחסכו
- מספר סשנים ו-handoffs

## הגדרות

```bash
ccso --config
```

ניתן לכבות/להדליק כל פיצ'ר:
- `smartRouting` — ניתוב מודל אוטומטי
- `translate` — תרגום עברית
- `codeCompression` — כיווץ קוד
- `gitContext` — הקשר git אוטומטי
- `secretScanner` — סורק מפתחות
- `memoryEnabled` — זיכרון בין סשנים
- `costThreshold` — סף עלות ל-handoff

---

## טיפים לחיסכון מקסימלי

1. **הרץ `ccso --init` בכל פרויקט חדש** — .claudeignore חוסך טוקנים עצומים
2. **שמור החלטות עם `/memory add`** — קלוד לא יצטרך לשאול שוב
3. **השתמש בתבניות** — פרומפטים ממוקדים = תשובות טובות יותר
4. **עשה `/handoff` בסוף כל יום** — מניע את הסשן הבא בצורה חלקה
5. **עבוד בבוקר (06:00–13:00 ישראל)** — שרתים פחות עמוסים
