🔌 פלטפורמות נתמכות
רק אינטגרציות עם קוד תמיכה ממשי ב-CCSO, יחד עם רמת התמיכה והמגבלות שלהן
חשוב: שעות עבודה בתוך Cursor או Windsurf לא יופיעו כחיסכון בדשבורד, כי CCSO לא רואה את הצ'אטים הפנימיים שלהם. הם נשארים כאן רק כאינטגרציות מסייעות, לא כמסלולי מדידה.
Measured Runtime
כאן נמצאות רק פלטפורמות שבהן CCSO גם יושב במסלול הריצה עצמו וגם יכול למדוד spend/savings בפועל.
Assisted Integrations
כאן CCSO יכול לעזור עם rules, MCP או helper מקומי, אבל לא מודד את השיחות של ה-IDE ולכן לא מציג חיסכון אמיתי מהן.
Instruction / Config Targets
אלה לא runtimes שבהם CCSO חוסך בזמן אמת, אלא יעדים לייצוא הוראות או קובצי פרויקט.
Utilities / Bridges
כלי עזר משלימים. שימושיים, אבל לא חלק ממסלול חיסכון של coding runtime.

Support Levels

Full backend
CCSO שולח את הפרומפט בעצמו, יודע להעריך עלות/חיסכון, ומספק חוויית dashboard/CLI מלאה.
Project rules + MCP
CCSO כותב קבצי כללים והגדרות MCP, אבל לא רואה את השיחות בתוך ה-IDE ולא מודד usage שלהן.
Project config only
CCSO יוצר קבצי פרויקט/סביבת עבודה, אבל לא שולט בריצה המקומית ולא מודד usage.
Instruction file only
CCSO יכול ליצור קובץ הוראות שצרכן אחר קורא. אין שליטה על האפליקציה עצמה ואין סטטיסטיקות שימוש.
Companion bridge
כלי עזר נפרד ש-CCSO מפעיל בפקודות ייעודיות. זה לא backend לעריכת קוד inline.

Not Shown On Purpose

Codex CLI, VS Code כללי, ו-Android Studio לא מוצגים כרגע בגריד התמיכה כי אין להם ב-CCSO אינטגרציה מאומתת וברורה ברמת מוצר.
המבחן כאן הוא לא "אפשר אולי לגרום לזה לעבוד", אלא "יש תמיכה אמיתית שאפשר להסביר למשתמש בלי להטעות".
⚙️ פיצ'רים פעילים
מה באמת נמדד, מה רק משפר workflow או reuse, ומה נשאר session hygiene כדי לא להטעות לגבי חיסכון

Quota / Session Hygiene
